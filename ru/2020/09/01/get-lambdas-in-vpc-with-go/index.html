Get Lambdas in Vpc With Go

Published Tue, Sep 1, 2020 by Maksym Postument

3 min

Language: English Ukrainian


Привет!

Сегодня я хочу рассказать как с помощью go можно получить список всех AWS Lambda которые находятся в VPC. Для начала нужно установить golang. Последние версию можно скачать с официального сайта.

После того как golang установлено создайте папку для скриптов mkdir awsscripts

Далее нужно инициализировать go.mod файл. Этот файл в котором хранятся все зависимости на стороние пакеты, и одновременно сделает go код модулем. И в будущем его можно будет устанавливать с помощью go get. Для создания модуля нужно вызвать команду go mod init github.com/mpostument/awsscripts. Где github… это имя модуля. Если вы использую другую систему контроля версий это может быть она, далее имя пользователя и название модуля.

После того как модуль создан нам потребуется aws-sdk для работы с aws. Установить его можно командой go get github.com/aws/aws-sdk-go@latest находясь в директории с go.mod. После этого в go mod добавится версия aws-sdk require github.com/aws/aws-sdk-go v1.34.14. В дальнейшем чтобы обновить версию aws-sdk можно вручную поменять версию в этом файле и запустить go mod tidy.

С настройкой завершено и можно перейти к коду. Создайте в корне папки awsscripts или в любой другой папке в середине awsscripts файл lambdaInVpc.go и откройте его.

Каждый go файл начинается с package. Поскольку нам нужно запускать go файл на прямую а не импортировать то package должно быть main.

Также нужно добавить список стороние модуля которые мы будем использовать

import (
	"fmt"
	"log"

	"github.com/aws/aws-sdk-go/aws/session"
	"github.com/aws/aws-sdk-go/service/lambda"
)

После списка импорта идет метод main, которий является входной точкой в ​​код. В main мы создадим aws client.

func main() {
    mySession := session.Must(session.NewSession())
    client := lambda.New(mySession)
    lambdas := getLambdaFunctions(client)
}

Здесь создается aws сессия и lambda client, и вызывается функция getLambdaFunctions в которую передается созданный client. Теперь приступим к написанию getLambdaFunctions.

func getLambdaFunctions(client *lambda.Lambda) []*lambda.FunctionConfiguration {
	input := &lambda.ListFunctionsInput{}

	var result []*lambda.FunctionConfiguration

	err := client.ListFunctionsPages(input,
		func(page *lambda.ListFunctionsOutput, lastPage bool) bool {
			result = append(result, page.Functions...)
			return !lastPage
		})
	if err != nil {
		log.Fatal("Not able to get lambdas", err)
		return nil
	}
	return result
}

Теперь разберем что здесь происходит. Для того чтобы получить список лямбда нужно вызвать функцию ListFunctions. Но поскольку эта метод возвращает только 50 первых лямбда она нам на подходит. Потому что если й 51 и более лямбда на aws акунти от всеравно вернет только первые 50. Но в aws есть метод с пагинацией ListFunctionsPages, и ее можно здесь использовать. Метод принимает два параметра первый параметры поиска ListFunctionsInput в которой можно задать регион, версию и еще несколько параметров. И вторая это метод пагинации.

В начале мы создаем две перемены это input типа ListFunctionsInput и result типа[]*lambda.FunctionConfiguration. Input будет передаваться в функцию ListFunctionsPages а result будет использоватся для хранения результата выполнения.

Далее идет сам вызов ListFunctionsPages. Эта метод возвращает только ошибку результат метода присваивается переменной err и дальше идет проверка не равна ли переменя nil. Если же метод вернул nil то происходит завершение программы и отображения ошибки.

В самой функции ListFunctionsPages происходит вызов еще одной функции которая добавляет результат выполнения в срез result. Вызов внутренней функции осуществляеться несколько раз пока мы не дойдем последней страницы. И в конце возвращаем срез result.

Теперь когда получили список всех лямбда нужно с них отфильтровать те которые в VPC. Для этого вернемся снова в метод main и добавить там итерации по всем лямбда с условием.

	for _, l := range lambdas {
		if l.VpcConfig != nil && len(l.VpcConfig.SubnetIds) > 0 {
			fmt.Println(*l.FunctionName)
		}
    }

Здесь мы итеруемось по результату выполнения getLambdaFunctions и проверяем VpcConfig функции не равен nil и или список subnet больше нуля. Две проверки нужны потому что для некоторых лямбд VpcConfig будет nil, а для других будет map в котором будут пустые списки. Например

<nil>
<nil>
{
  SecurityGroupIds: [],
  SubnetIds: [],
  VpcId: ""
}
<nil>

Теперь можно запустить код и получить результат.
go run lambdaInVpc.go
