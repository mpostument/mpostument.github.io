Привіт!

В цій статі я розкажу як в terraform можна використовувати статичні файли або ж генерувати файли динамічно. Покажу це на прикладі user_data в EC2. За допомогою user_data можна передати скрипт який виконається на сервері в момент створення. Цей файл можна мати одразу готовий і передавати за допомогою terraform в user_data, або ж контент можна генерувати динамічно і передавати в user_data.

Terraform file⌗

Створю файл з назвою setup_httpd.sh в якому буде наступний контент

#!/bin/bash
yum install httpd -y
/sbin/chkconfig --levels 235 httpd on
service httpd start
echo "<h1>Maksym Website</h1>" > /var/www/html/index.html

Цей скрипт встановить httpd і створить html сторінку. Тепер як я можу використати цей файл? Візьму terraform код з попередньої статті і добавлю в нього user_data.

resource "aws_instance" "foo" {
  ami                    = "ami-0cff7528ff583bf9a"
  instance_type          = "t2.micro"
  subnet_id              = "subnet-222a93327f9a744ed"
  vpc_security_group_ids = ["sg-2220a119757753b6e"]
  user_data              = file("setup_httpd.sh")
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

І тепер запущу terraform. І як сервер створиться в AWS console можна подивитись скрипт який було згенеровано.

І якщо відкрити IP address сервера в браузері то побачу

Terraform Template⌗

Але цей файл був статичний і якщо потрібно змінити наприклад контент вебсторінки файл потрібно міняти, але в terraform є метод templatefile() який дозволяє генерувати контент файлу динамічно. Для цього створю новий файл який буде називатись setup_httpd.sh.tpl якому контент буде майже такий самий як і в попередньою але з однією різницею. В останньому рядку замість тексту Maksym Website, я використовую ${user} Website. Де ${user} це параметр який terraform замінить на відповідне значення.

#!/bin/bash
yum install httpd -y
/sbin/chkconfig --levels 235 httpd on
service httpd start
echo "<h1>${user} Website</h1>" >/var/www/html/index.html

Також потрібно і змінити сам код terraform. В цьому випадку виклик методі file я заміню на templatefile і окрім самого файлу передам параметр user.

resource "aws_instance" "foo" {
  ami                    = "ami-0cff7528ff583bf9a"
  instance_type          = "t2.micro"
  subnet_id              = "subnet-222a93327f9a744ed"
  vpc_security_group_ids = ["sg-2220a119757753b6e"]
  user_data = templatefile("setup_httpd.sh.tpl", {
    user = "Jhon"
  })
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

Так само як і попереднього разу гляну згенерований скрипт
