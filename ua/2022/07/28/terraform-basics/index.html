Частина 1 - Terraform Install

Привіт!

В попередній статті ми розглянули як встановити terraform. А зараз розглянемо як за допомогою terraform можна створювати та видаляти ресурси в AWS.

Для того щоб terraform міг керувати ресурсами в AWS потрібно встановити terraform AWS provider. Terraform зробить це автоматично якщо я вкажу використовувати AWS provider в terraform коді і викличу terraform init.

Для цього я створю директорію terraform_code і в ній створю файл provider.tf. Назва файлу може будь якою але якщо він називається provider.tf то одразу стає зрозумілим що в цьому файлі знаходиться

mkdir terraform_code
touch provider.tf

Добавлю наступний контент в provider.tf

terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
      version = "4.23.0"
    }
  }
}

В секції terraform вказано який provider використовувати та яку версію. Інформацію про версії можна знайти в GitHub.

Як файл збережено одразу можна виконати terraform init для того щоб завантажити provider. Але перед тим потрібно вибрати яка версія terraform буде використовуватись. Для цього створю файл з назвою .terraform-version і в нього запишу 1.2.6 (версія terraform). Далі tfenv зробить все автоматично.

В цьому файлі будуть вказані усі версію залежностей які використовує terraform і при наступних запусках terraform init будуть використані версії які вказані в файлі. В директорії .terraform будуть знаходитись завантаженні модулі та провайдери

|-- .terraform
|   `-- providers
|       `-- registry.terraform.io
|           `-- hashicorp
|               `-- aws
|                   `-- 4.23.0
|                       `-- linux_amd64
|                           `-- terraform-provider-aws_v4.23.0_x5

Також для того щоб керувати ресурсами потрібно в AWS для свого користувача згенерувати Access Key та Secret Key і зберегти їх в в файлі ~/.aws/credentials

[default]
aws_access_key_id = ACCESS_KEY
aws_secret_access_key = SECRET_KEY

Приступимо до написання коду. Для цього створю файл з назвою main.tf який має знаходитись в тій самій директорії що і provider.tf. Наступний контент можна додати як і в provider.tf так і в main.tf або створити взагалі інший файл наприклад terraform.tf. Я добавлю в файл provider.tf

provider "aws" {
  region              = "us-east-1"
  profile             = "default"
}

В якому вказую який регіон використовувати та який aws profile використовуємо. Декілька кроків вище я створив profile з назвою default. В одному terraform коді може використовуватись декілька profile які будуть використовувати різні регіони або різні AWS акаунти.

В main.tf додам наступний контент

resource "aws_instance" "foo" {
  ami                    = "ami-0cff7528ff583bf9a"
  instance_type          = "t2.micro"
  subnet_id              = "subnet-222a93327f9a744ed"
  vpc_security_group_ids = ["sg-2220a119757753b6e"]
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

Цей код описує з якими параметрами створити ec2 instance. Я вказую яку AMI використовувати (Amazon Linux 2) в якому subnet та з якою security group створювати сервер. Також зазначаю які теги має мати сервер та диск.

Команда terraform plan -out tf.plan проаналізує що має бути створено і виведе це на екран. Частина значень буде вказано як наприклад + ami = "ami-0cff7528ff583bf9a" решта будуть мати значення user_data = (known after apply), це означає що на даний момент terraform ще не знає які будуть значення цих параметрів і вони будуть відомі після того як ресурс буде створено. За допомогою ключа -out tf.plan результат буде збережено в файл. І якщо те що вивів terraform потрібно запустити terraform apply "tf.plan" і відповідні ресурси будуть створенні.

Для того щоб видалити створенні ресурси можна запустити terraform destroy
