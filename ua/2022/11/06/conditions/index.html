Привіт!

Розглянемо умови в Game Maker. Умови можна створювати як за допомогою if/else так і switch. Розглянемо обидва варіанти.

If/Else⌗

В Game Maker if/else умови такі ж як і в інших мовах програмування. Сам блок виглядає так:

if (<expression>)
{
    <statement>;
}
else
{
    <statement>;
}

Наприклад ми можемо мати на перевірку здоров’я нашого об’єкта. Наприклад якщо hp < 100 то знищуємо об’єкт. Для того щоб це зробити використаємо новий типу івенту який називається Step. Цей event виконується кожен кадр. Зазвичай основна логіка відбувається саме в цьому івенті.

Спочатку в event Create задамо початкову кількість hp

hp = 100

Тоді додамо event Step і в ньому додамо наступний код. В дужках біля if є і сама наша умова. Де ми перевіряємо чи hp є менше рівне 100. Окрім <= менше рівно є ще < менше, > більше, = рівне, нерівне !=, >= більше рівне. Також можна одразу використовувати не одну умову а більше, для цього є && що означає and та || що означає or. В коді можна записувати як і символами так і буквами. І це може виглядати так if (hp <= and armor <= 0)

Якщо ж у нас умова повертає False тому що кількість hp є більшою за 0 ми переходимо в else. В else ми просто переміщуємо наш об’єкт по діагоналі і віднімаємо 1 hp. If умова перевіряється кожен кадр і вже через декілька секунд кількість hp стане меншою за 0 і об’єкт знищиться.

if (hp <= 0) {
	instance_destroy()
}
else { 
	x = x + 1
	y = y + 1
	hp = hp - 1
}

Якщо нам потрібно задати декілька умов то можна використати else if. Наприклад якщо залишається менше 50 hp додати ще один об’єкт

if (hp <= 0) {
	instance_destroy()
}

else if (hp <= 50) {
	instance_create_layer(x, y, "Instances", obj_vars)
}
else { 
	x = x + 1
	y = y + 1
	hp = hp - 1
}

Якщо умов стає багато то в такому випадку краще використати switch.

Switch⌗

Ось так виглядає switch

switch (<expression>)
{
    case <constant1>:
        <code>
    break;

    case <constant2>:
        <code>
    break;
    
    // more cases (with breaks)

    default:
        <code>;
}

Наприклад ми можемо виконувати якусь дію в залежності від того яка клавіша натиснута. В дужках біля switch вказуємо keyboard_key яка є вбудованим параметром який приймає значення в залежності від того яка клавіша натиснута. І тоді описуємо кожен case. Якщо натиснута клавіша вверх переміщуємо героя вверх, якщо вниз то вниз і так далі

switch (keyboard_key)
{
    case vk_left:
        x -= 4;
    break;

    case vk_right:
        x += 4;
    break;

    case vk_up:
        y -= 4;
    break;

    case vk_down:
        y += 4;
    break;
}

Також в case може буде дефолтна дія, вона виконується в тому випадку якщо всі інші кейси повернули false.
