Привіт!

В цьому пості розглянемо роботу з масивами в Game Maker.

Масиви можуть містити в собі багато елементів і бувають одномірними та багатомірними. На відміну інших мов програмування масив в Game Maker не потребує ініціалізації. Таким чином можна додати елемент в масив, де спочатку йде назва масиву а в квадратних дужках вказується його порядковий номер. Нумерація починається з нуля

inventory[0] = "bow"
inventory[1] = "arrow"

В результаті це буде виглядати як масив з двох елементів [“bow”, “arrow”]. Також часто використовуються двомірні масиви. Для цього потрібно додати ще одні квадратні дужки які будуть означати колонку

inventory[0][0] = "bow"
inventory[0][1] = "arrow"
inventory[1][0] = "sword"
inventory[1][0] = "shield"

В цьому випадку це буде виглядати як таблиця з двох рядків та двох колонок.

For loop⌗

Для наповнення масиву даними чи читанням з нього часто використовується цикл for. Це значно зменшує кількість коду яку потрібно написати. Наприклад якщо потрібно додати 100 елементів в масив то можна 100 раз скопіювати рядок присвоєння елемента масиву але працювати з цим не дуже зручно. Створимо пустий масив з 100 елементів.

inventory[0] = "empty"
inventory[1] = "empty"
inventory[98] = "empty"
...
inventory[99] = "empty"

Якщо ж використати for це буде значно компактніше. Цикл for виглядає таким чином

for (i = 0; i < 99, i++) {
    # CODE
}

i вказує звідки ми починаємо відлік. Далі йде умова що i має бути менше 99, як тільки і стане 99 цикл одразу закінчиться. І в кінці маємо i++ що означає що і буде збільшуватись на 1. Таким чином коли починається цикл ми маємо i = 0. Після першого проходження циклу i збільшується на 1 і стає рівним i = 1. І так аж доки i не стане 99. Далі цикл завершується.

Спробуємо створити пустий одновимірний масив за допомогою циклу.

for (i = 0; i < 99, i++) {
    inventory[i] = "empty"
}

Таким чином при кожному проходженні циклу i буде збільшуватись і ми створимо масив з 100 елементів з значенням “empty”. Це можна додати в event Draw щоб відобразити масив. Але перед тим він має бути створений. Його створення в викликаю в івенті Create. В draw_text до координати x додам ще 70*i, це для того щоб був відступ по x координаті між елементами масиву. Коли i = 0 то 70*i теж буде 0. Далі це буде 70, 140 і так далі.

for (i = 0; i < 99, i++) {
    draw_text(10+70*i, 10, inventory[i])
}

Двовимірний масив можна створити таким ж чином тільки нам буде потрібно два цикли

for (i = 0; i < 99, i++) {
    for (j = 0; j < 99, i++) {
        inventory[i][j] = "empty"
    }
}

Що буде відбуватись в цьому випадку? Спочатку починаємо перший цикл де i = 0, далі одразу попадаємо в другий цикл де j = 0. І в масив по індексу [0][0] робимо перший запис. Далі i в нас так і залишається 0 а j збільшується на 1. Тому в масив ми робимо запис в індекс [0][1] і так далі аж доки j не стане 99. Тоді вкладений цикл завершиться і ми повернемось до циклу з i. Тут i збільшиться на 1 і знову піде в вкладний цикл де j рівне 0. І зробимо запис в індекс [1][0] і знову доки j не стане 99 а i збільшится на 1 аж доки не дійде до 99 і тоді цикл завершиться.

Для того щоб відобразити двовимірний масив теж буде потрібно два цикли. А все решта так само як і з одновимірним

for (i = 0; i < 99, i++) {
    for (j = 0; j < 99, i++) {
        draw_text(10+70*i, 10+70*j, inventory[i][j])
    }
}
