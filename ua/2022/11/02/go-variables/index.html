Привіт!

Сьогодні поговоримо про параметри в golang. Є три варіанти як можна створити параметри розглянемо кожен з них.

Var keyword⌗

Перший варіант це створення параметра за допомогою ключового слова var. Так можна створити параметр який не буде мати значення (буде мати початкове значення для кожного типу). Виглядає це так var PARAM_NAME PARAM_TYPE наприклад var level int

package main

import "fmt"


func main() {
    var level int
	fmt.Println(level)
}

$ go run main.go
0

Навіть яко ми не вказала значення для параметра level, він всерівно буде мати значення. Тепер коли параметр створено, йому можна змінити значення.

package main

import "fmt"

func main() {
	var level int
	level = 10
	fmt.Println(level)
}

$ go run main.go
10

Також цей параметр можна винести за межі функції і код буде так само працювати.

package main

import "fmt"

var level int

func main() {
	level = 10
	fmt.Println(level)
}

Якщо потрібно створити декілька параметрів одночасно це можна зробити так для того щоб не повторювати слово var

var (
	level int
	exp   int
	life  int
)

Var keyword with value⌗

Також коли створюємо параметр йому можна задати початкове значення. var PARAM_NAME PARAM_TYPE = PARAM_VALUE, наприклад var level int = 10.

package main

import "fmt"

func main() {
	var level int = 10
	fmt.Println(level)
}

$ go run main.go
10

Оскільки ми вказуємо одразу значення параметра то тип можна опустити var level = 10 і робота програми не зміниться.

package main

import "fmt"

func main() {
	var level = 10
	fmt.Println(level)
}

$ go run main.go
10

Цей параметр так само як і попередній можна винести за межі функції і код буде працювати. Якщо потрібно створити декілька параметрів одночасно це можна зробити схожим способом як і попередньо

var (
	level = 10
	exp   = 300
	life  = 4
)

Parameters without var⌗

Третій варіант створення параметра це level := 10. Тут вже не використовується слово var, але такі параметри можна створювати тільки всередині функцій

package main

import "fmt"

func main() {
	level := 10
	fmt.Println(level)
}

$ go run main.go
10

А якщо спробую винести параметр за межі функції то отримаю наступне

go run main.go
# command-line-arguments
./main.go:5:1: syntax error: non-declaration statement outside function body

Variables Naming⌗

Зазвичай в golang параметри називають наступним чином. Якщо параметр починається не з великої літери це означає що він доступний тільки в межах цього пакету.

var level = 1

Якщо ж параметр починається з великої літери то цей параметр стає доступний для використання в інших пакетах

var Level = 1

Якщо параметр складається з більше ніж одного слова то його назва теж залежить чи він має бути доступний тільки всередині пакета чи і з інших. Друге слово в назві параметра завжди має починатись з великої літери

var heroLevel = 1
var HeloLevel = 1

Variables conversion⌗

Якщо у нас є параметр одного типу його можна конвертувати в інший тип. Наприклад в нас є int на потрібно float. Це можна здійснити наступним чином

package main

import "fmt"

func main() {
	var level int = 10
	levelFloat := float64(level)
	fmt.Printf("%v: %T", levelFloat, levelFloat)
}

go run main.go
10: float64

Спробуємо тепер зробити таке саме але з типом string

package main

import "fmt"

func main() {
	var level int = 123
	levelFloat := string(level)
	fmt.Printf("%v: %T", levelFloat, levelFloat)
}

go run main.go
{: string

Результат не зовсім той що ми очікували. При конвертуванні golang бере unicode значення під цим номером. І під номером 123 якраз знаходиться значення {.

То як ж конвертувати int в string. Для цього в go є спеціальні методи в пакеті strconv, наприклад strconv.Itoa для того щоб конвертувати int в string

package main

import (
	"fmt"
	"strconv"
)

func main() {
	var level int = 123
	levelFloat := strconv.Itoa(level)
	fmt.Printf("%v: %T", levelFloat, levelFloat)
}

$ go run main.go
123: string

Shadowing⌗

Якщо в нас є параметр глобальний за межами функції і ми створюємо параметр з такою самою назвою в функції це називається shadowing. Розглянемо на прикладі

package main

import (
	"fmt"
)

var level int = 1

func main() {
	fmt.Println(level)
	var level = 20
	fmt.Println(level)
	printLevel()
}

func printLevel() {
	fmt.Println(level)
}

Тут в нас є параметр level з значенням один ззовні функції main, ми виводимо цей параметр і тоді створюємо ще один параметр з такою ж назвою всередині функції. І виводимо його значення. Також в нас є ще один метод printLevel який також виводить значення параметра level.

$ go run main.go
1
20
1

Як видно з результату спочатку відбувся вивід level глобального, потім вивід локального параметра з функції і в кінці знову глобальний параметр
