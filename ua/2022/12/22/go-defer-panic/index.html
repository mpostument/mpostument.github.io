Defer, Panic and Recovery in Golang

Привіт!

В цьому пості розглянемо defer, panic, recovery

Defer⌗

Defer дозволяє відкласти виконання коду і виконати його в той момент коли буде вихід з функції

package main

import "fmt"

func main() {
	defer fmt.Println("Hello!")
	fmt.Println("Bye!")
}

$ go run main.go
Bye!
Hello!

Хоча по коду спочатку мав вивестись напис Hello!, першим вивівся напис Bye! і тільки тоді Hello!. Це тому що ми відклали виконання цього коду до моменту виходу з функції main.

Найчастіше defer використовується щоб очистити ресурси після використання. Наприклад закрити підключення до бази даних чи файл після використання. Звичайно це можна зробити і без defer, але в такому випадку потрібно не забути закрити підключення після завершення роботи, і закриття буде знаходитись від відкриття

package main

import (
	"io"
	"log"
	"os"
)

func main() {
	file, err := os.Create("file.txt")
	if err != nil {
		log.Fatalln(err)
	}
	_, err = io.WriteString(file, "Writing text to file")
	if err != nil {
		log.Fatalln(err)
	}
	file.Close()
}

Тут ми закриваємо файл за допомогою file.Close() в кінці коду який працює з файлом. Якщо ми додамо defer то закриття можна додати одразу після відкриття. Але викликане воно буде так само як і в першому варіанті

package main

import (
	"io"
	"log"
	"os"
)

func main() {
	file, err := os.Create("file.txt")
	defer file.Close()
	if err != nil {
		log.Fatalln(err)
	}
	_, err = io.WriteString(file, "Writing text to file")
	if err != nil {
		log.Fatalln(err)
	}
}

Якщо ми маємо декілька defer то вони будуть викликані в зворотньому порядку

package main

import (
	"fmt"
)

func main() {
	defer fmt.Println(1)
	defer fmt.Println(2)
	defer fmt.Println(3)
}

$ go run main.go
3
2
1

Panic⌗

panic використовується у випадку якщо сталась помилка і потрібно зупинити виконання програми

package main

import (
	"fmt"
)

func main() {
	fmt.Println(1)
	panic("Critical Error")
	fmt.Println(2)
}

$ go run main.go
1
panic: Critical Error

goroutine 1 [running]:
main.main()
        /Users/maksym.postument/go/src/github.com/mpostument/hello/main.go:9 +0x6c
exit status 2

Також panic відбудеться якщо ми спробуємо поділити на нуль

package main

import (
	"fmt"
)

func main() {
	fmt.Println(2/0)
}

$ go run main.go
# command-line-arguments
./main.go:8:16: invalid operation: division by zero

Recover⌗

recover дозволяє відновитися після panic і не завершувати роботу програми. Для цього ми зробимо окрему функцію яка буде називатись handlePanic, в які буде викликатись recover(). І її будемо викликати за допомогою defer в методі в якому може трапитись panic.

package main

import (
	"fmt"
)

func handlePanic() {

	a := recover()

	if a != nil {
		fmt.Println("RECOVER", a)
	}
}

func main() {
	fmt.Println("Start execution")
	runPanic()
	fmt.Println("End of execution")
}

func runPanic() {
	defer handlePanic()
	panic("Critical Error")
}

$ go run main.go
Start execution
RECOVER Critical Error
End of execution

Після того як виник panic. Виконання програми всерівно продовжилось не зважаючи на panic
