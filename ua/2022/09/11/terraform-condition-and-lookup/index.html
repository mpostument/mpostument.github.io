В цьому пості розкажу про condition та lookups.

Conditions⌗

Умови в terraform використовуються для того щоб вибрати одне значення в залежності від іншого. Наприклад якщо це дев середовище то використовувати тип сервера t3.micro а якщо продакшин то m5.large. Це легко зробити за допомогою умови. Записуються умови так condition ? true_val : false_val. І якщо нам потрібно зробити умову на тип середовища то це буде мати такий вигляд var.env == "prod" ? "m5.large" : "t3.micro". Тут перевіряю чи параметр env має значення prod, і якщо так то використовую m5.large, в усіх ж інших випадках це t3.micro. Повний код створення ec2 буде виглядати так.

variable "env" {
  default     = "dev"
  description = "Environment name"
  type        = string
}

resource "aws_instance" "foo" {
  ami                    = data.aws_ami.amzn.id
  instance_type          = var.env == "prod" ? "m5.large" : "t3.micro"
  subnet_id              = "subnet-5f70bf74"
  vpc_security_group_ids = ["sg-b04b8cd4"]
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

Також за допомогою умов можна вказувати чи створювати ресурс чи ні. В попередньому прикладі для dev середовища я використовував менший розмір сервера, але в деяких випадках може бути потрібно для dev цього ресурсу взагалі не створювати. Для цього потрібно додати атрибут count і в ньому вказати - якщо prod то створити один сервер а якщо не prod то 0

resource "aws_instance" "foo" {
  count                  = var.env == "prod" ? 1 : 0
  ami                    = data.aws_ami.amzn.id
  instance_type          = "m5.large"
  subnet_id              = "subnet-5f70bf74"
  vpc_security_group_ids = ["sg-b04b8cd4"]
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

$ terraform plan
data.aws_ami.amzn: Reading...
data.aws_ami.amzn: Read complete after 1s [id=ami-05fa00d4c63e32376]

No changes. Your infrastructure matches the configuration.

Тепер для того щоб звернутись потрібно вказати індекс. Оскільки я створюю тільки один сервер то індекс в мене буде 0, і звернення буде виглядати так aws_instance.foo[0].private_ip. Якщо ж створюється більше ніж один сервер то до них можна буде звертатись по їх індексах 1,2,3 і так далі.

Lookup⌗

З умовами я міг вказати що якщо prod то один тип сервера а якщо ні то інший. А що робити якщо в нас декілька середовищ як dev, test, staging, prod і для кожного з них потрібні різні типи серверів. З цим нам допоможе lookup. Lookup функція дозволяє отримати значення з map в залежності від значення ключа.

Для початку потрібно створити map яка буде мати значення типу сервера до середовища.

variable "env" {
  default     = "dev"
  description = "Environment name"
  type        = string
}

variable "instance_types" {
  type = map(string)
  default = {
    "prod"    = "m5.xlarge",
    "staging" = "m5.large",
    "test"    = "t3.large",
    "dev"     = "t3.micro"
  }
}

Тепер це можна використати в aws_instance коді. Для lookup функції потрібно три аргументи. Перший map яка містить значення, другий це значення ключа по якому робимо пошук і третій це дефолтне значення на випадок якщо потрібно ключа в map не буде

resource "aws_instance" "foo" {
  count                  = var.env == "prod" ? 1 : 0
  ami                    = data.aws_ami.amzn.id
  instance_type          = lookup(var.instance_types, var.env, "t3.micro")
  subnet_id              = "subnet-5f70bf74"
  vpc_security_group_ids = ["sg-b04b8cd4"]
  tags = {
    Env = "Dev"
  }
  volume_tags = {
    "Env" = "Dev"
  }
}

І тепер яке б значення env параметра я б не передав потрібний тип сервера буде використано.
