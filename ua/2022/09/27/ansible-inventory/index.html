Частина 1 - Manage Ansible Versions With pyenv

Привіт!

Сьогодні розкажу що таке inventory в ansible так яких типів він буває.

В ansible є два типи inventory - це static та dynamic. Static використовується коли ми на перед знаємо список серверів на яких має відпрацювати ansible. А dynamic коли цих даних наперед ми не маємо і отримуємо динамічно.

Static inventory⌗

Щоб використовувати static inventory достатньо просто створити файл наприклад inventory і в ньому можна вказувати ip адрес наприклад:

192.168.1.1

І його одразу ж можна використовувати вказавши цей файл в ansible команді за допомогою -i inventory all. all означає запустити ansible на всі сервери з inventory файлу.

$ ansible -m ping -i inventory all --private-key SSH_KEY --user SSH_USER
192.168.1.1 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

Також сервери можна об’єднувати в групи.

[web]
192.168.1.1

[database]
192.168.1.1

І тепер можна запускати ansible тільки для якоїсь групи. Для цього потрібно замість all вказати назву групи. Якщо потрібно запусти для усіх серверів то можна знову ж вказати all.

$ ansible -m ping -i inventory web --private-key SSH_KEY --user SSH_USER
192.168.1.1 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

Також в inventory для кожного сервера або групи можна вказувати набір параметрів. Наприклад значення яке ми передавали за допомогою ключа –user можна вказати в inventory.

[web]
192.168.1.1 ansible_user=centos

Тепер ключ –user можна не додавати

$ ansible -m ping -i inventory web --private-key SSH_KEY
192.168.1.1 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

Також параметри можна задати на рівні групи, для цього потрібно додати ще одну групу з назвою [GROUP_NAME:vars]

[web]
192.168.1.1

[web:vars]
ansible_user=centos

Результат буде такий ж як і в попередньому варіанті, тільки тепер ці параметри будуть застосовані до цієї групи.

Як параметри можуть передаватись будь які значення які в подальшому будуть використовуватись в плейбуках.

Також декілька груп можна додавати в одну групу.

[web]
192.168.1.1

[db]
192.168.1.2

[webdb:children]
web
db

І якщо при запуску ansible я вкажу -i inventory webdb, ansible буде запущено для усіх серверів з цих двох груп.

Group Vars⌗

Inventory не краще місце для зберігання параметрів. Краще для цього створити теку з назвою group_vars і в ній організувати параметри для кожної групи. Групу можна організувати як текою так і файлом. В цьому прикладі я створив теку web яка відповідає назві групи а всередині файл який може мати будь яку назву.

tree group_vars
group_vars
└── web
    └── web

В файлі web тепер можна вказувати параметри які будуть стосуватись всієї групи web

ansible_user: centos

І якщо я запущу ansible то знову отримаю результат як і попереднього разу.

Dynamic Inventory⌗

Dynamic inventory потрібен якщо нам наперед не відомі IP адреси наперед. Наприклад з AWS де сервери можуть часто мінятись і мати різні IP адреси. Dynamic inventory це скрипт який передається параметру -i і при запуску цей скрипт отримаю всю потрібну інформацію про сервери. Деякі dynamic inventory стали частиною ansible і їх уже не потрібно додатково завантажувати, а достатньо просто вказати що його потрібно використати.

Для того щоб використати AWS dynamic inventory потрібна додаткова python бібліотека boto3. Її можна встановити за допомогою pip install boto3.

Оскільки AWS dynamic inventory є частиною ansible, ніяких скриптів завантажувати не потрібно. Але потрібно створити конфігураційний файл aws_ec2.yml. Найпростіший буде виглядати так

plugin: aws_ec2
regions:
  - REGION
boto_profile: AWS_PROFILE

В такому випадку ansible буде запущено на усі сервери які є в даному AWS акаунті в вказаному регіоні. Можна використовувати фільтри для того щоб відфільтрувати тільки потрібні сервери. Наприклад тільки сервери в яких є тег type = database.

plugin: aws_ec2
regions:
  - REGION
boto_profile: AWS_PROFILE
filters:
  tag:type: database

Фільтри можна комбінувати один з одним. Всі можливі опції можна знайти в документації конкретного dynamic inventory(AWS docs).
