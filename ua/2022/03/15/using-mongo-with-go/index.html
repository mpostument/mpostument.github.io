Using Mongo With Go

2022-03-15 :: Maksym Postument :: 3 min read (524 words)
#Go  #Mongo  #Beginner 

Привіт!

Сьогодні подивимось як використовувати go з mongo. І розпочнемо з запуску mongo. Я буду використовувати docker-compose. Для цього потрібно створити файл з назвою docker-compose.yml. В файлі потрібно вказати яку версію монги використовувати. Я не вказую нічого це означає що буде остання версія. Також потрібно вказати ім’я користувача та пароль і порт який буде відкритий для доступу

version: '3.1'

services:

  mongo:
    image: mongo
    restart: always
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: password
    ports:
      - 27017:27017

Після того як файл збережено потрібно виконати команду docker-compose up -d (Docker та docker-compose має бути встановлено) і буде запущено контейнер з mongo.

З конфігурацією монго завершено, і можна приступити до коду. Ініціалізуємо go модуль і звантажимо драйвер

go mod init
go get go.mongodb.org/mongo-driver/mongo

Створимо main.go і приступимо до кодінгу. Розпочнемо з методу mongoInsert який буде записувати дані в базу. В методі викликається mongo.Connect в який передається контекст та параметри. Як параметри я передаю посилання на монго в форматі mongodb://username:password@mongoAddress:mongoPort. Також створюю context з тайм-аутом, щоб автоматично завершувати запити якщо вони тривають довше чим потрібно. Для того, щоб здійснити запит потрібно створити структуру яка буде мати поля такі які мають бути записані. В мене це структура User з двома полями Name i Surname типу string. До кожного з полів має бути додана анотація в якій буде вказано як поле буде називатись в базі даних. Методі для запису InsertOne вертає монго ID. Який буде виведено після успішного запису. База даних та колекція автоматично будуть створенні після операції запису.

package main

import (
	"context"
	"log"
	"time"

	"go.mongodb.org/mongo-driver/mongo"
	"go.mongodb.org/mongo-driver/mongo/options"
)

type User struct {
	Name    string `bson:"name"`
	Surname string `bson:"surname"`
}

func main() {
	err := mongoInsert()
	if err != nil {
		log.Fatalln(err)
	}
}

func mongoInsert() error {
	ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
	defer cancel()
	client, err := mongo.Connect(ctx, options.Client().ApplyURI("mongodb://root:password@127.0.0.1:27017"))
	if err != nil {
		return err
	}
	defer client.Disconnect(ctx)

	collection := client.Database("mpostument").Collection("users")
	user := User{
		Name:    "Maksym",
		Surname: "Postument",
	}

	res, err := collection.InsertOne(ctx, user)
	if err != nil {
		return err
	}
	id := res.InsertedID
	log.Printf("Insert ID, %s", id)
	return nil

}

Тепер можна підключитись до бази даних і подивись чи є записи. Для цього потрібно зайти в докер контейнер

docker ps
# Скопіювати CONTAINER ID і виконати
docker exec -it a61578b802e7 bash

Тепер викличемо консольний кієнт

mongo -u root -p password

Підключимось до створеної бази даних

> use mpostument
switched to db mpostument

І отримати дані з бази

> db.users.findOne()
{
	"_id" : ObjectId("6231a26576e5effdd561a9d8"),
	"name" : "Maksym",
	"surname" : "Postument"
}

А тепер подивимось як можна прочитати дані з монги за допомогою golang. Початок методу такий самий, створюється клієнт і підключається до колекції. Далі створюємо змінну user і викликаємо метод FindOne в який передаю контекст і фільтр за яким робити пошук. В моєму варіанті шукаємо об’єкт в якого поле name рівне Maksym.

func main() {

	err = mongoGet()
	if err != nil {
		log.Fatalln(err)
	}
}

func mongoGet() error {
	ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
	defer cancel()
	client, err := mongo.Connect(ctx, options.Client().ApplyURI("mongodb://root:password@127.0.0.1:27017"))
	if err != nil {
		return err
	}
	defer client.Disconnect(ctx)
	collection := client.Database("mpostument").Collection("users")

	var user User

	res := collection.FindOne(ctx, bson.D{{"name", "Maksym"}})
	res.Decode(&user)
	fmt.Println(user)
	return nil
}

Також можна замість FindOne виконати метод Find який буде вертати багато об’єктів.
