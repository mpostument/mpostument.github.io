Get Lambdas in Vpc With Go

Published Tue, Sep 1, 2020 by Maksym Postument

3 min

Language: English Russian


Привіт!

Сьогодні я хочу розказати як за допомогою go можна отримати список всіх AWS Lambda які знаходяться в VPC. Для початку потрібно встановити golang. Останю версію можна завантажити з офіційного сайту.

Після того як golang встановлено створіть папку для скриптів mkdir awsscripts

Далі потрібно ініціалізувати go.mod файл. Цей файл в якому зберігаються всі залежності на стороні пакети, і одночасно зробить go код модулем. І в майбутньому його можна буде встановлювати за допомогою go get. Для створення модуля потрібно викликати команду go mod init github.com/mpostument/awsscripts. Де github… це імя модуля. Якщо ви використовую іншу систему контролю версій це може бути вона, далі ім’я користувача та назва модуля.

Після того як модуль створено нам буде потрібен aws-sdk для роботи з aws. Встановити його можна командою go get github.com/aws/aws-sdk-go@latest знаходячь в директорії з go.mod. Після цього в go mod добавится версія aws-sdk require github.com/aws/aws-sdk-go v1.34.14. Надалі щоб оновити версію aws-sdk можна вручну поміняти версію в цьому файлі та запустити go mod tidy.

З налаштуванням завершено і можна перейти до коду.
Створіть в корені папки awsscripts або ж в будь якій інші папці в середині awsscripts файл lambdaInVpc.go і відкрийте його.

Кожен go файл починається з package. Оскільки нам потрібно запускати go файл на пряму а не імпортувати то package має бути main.

Також потрібно добавити список стороніх модулів які ми будемо використовувати

import (
	"fmt"
	"log"

	"github.com/aws/aws-sdk-go/aws/session"
	"github.com/aws/aws-sdk-go/service/lambda"
)

Після списку імпортів іде метод main, який є вхідною точкою в код. В main ми створимо aws client.

func main() {
    mySession := session.Must(session.NewSession())
    client := lambda.New(mySession)
    lambdas := getLambdaFunctions(client)
}

Тут створюється aws сесія і lambda client, і викликається функці я getLambdaFunctions в яку передається щойно створений client. Тепер приступимо до написання getLambdaFunctions.

func getLambdaFunctions(client *lambda.Lambda) []*lambda.FunctionConfiguration {
	input := &lambda.ListFunctionsInput{}

	var result []*lambda.FunctionConfiguration

	err := client.ListFunctionsPages(input,
		func(page *lambda.ListFunctionsOutput, lastPage bool) bool {
			result = append(result, page.Functions...)
			return !lastPage
		})
	if err != nil {
		log.Fatal("Not able to get lambdas", err)
		return nil
	}
	return result
}

Тепер розберемо що тут відбувається. Для того щоб отримати список лямбд потрібно викликати функцію ListFunctions. Але оскільки ця метод повертає тільки 50 перших лямбд вона нам на підходить. Тому що якщо ї 51 і більше лямбд на aws акунті од всерівно поверне тільки перші 50. Але в aws є метод з пагінацією ListFunctionsPages, і її можна тут використати. Метод приймає два параметра перший параметри пошуку ListFunctionsInput в якій можна задати регіон, версію та ще декілька параметрів. І друга це метод пагінації.

На початку ми створюємо дві зміні це input типу ListFunctionsInput і result типу []*lambda.FunctionConfiguration. Input буде передаватись в функцію ListFunctionsPages а result буде використуватись для зберігання результату виконання.

Далі йде сам виклик ListFunctionsPages. Ця метод вертає тільки помилку результат цієї метод присвоюється змінній err і далі йде первірка чи ця зміна не рівна nil. Якщо ж метод вернула nil то відбувається завершення програми і відображення помилки.

В самій функції ListFunctionsPages відбувається виклик ще однієї функції яка добавляє результат виконання в зріз result. Виклик внутрішньою функції здіюснюється декілька раз доки ми не дійдемо останьої сторінки. І в кінці вертаємо зріз result.

Тепер коли отримали список всіх лямбд потрібно з них відфільтрувати ті які в VPC. Для цього повернемось знову в метод main і добавити там ітерації по всіх лямбдах з умовою.

	for _, l := range lambdas {
		if l.VpcConfig != nil && len(l.VpcConfig.SubnetIds) > 0 {
			fmt.Println(*l.FunctionName)
		}
    }

Тут ми ітеруємось по результату виконання getLambdaFunctions і перевірємо чи VpcConfig функції не рівний nil і чи список subnet більше за нуль. Дві перевірки потрібні тому що для деяких лямбд VpcConfig буде nil, а для інших буде map в якому будуть пусті списки. Наприклад

<nil>
<nil>
{
  SecurityGroupIds: [],
  SubnetIds: [],
  VpcId: ""
}
<nil>

Тепер можна запустити код і отримати результат.
go run lambdaInVpc.go
